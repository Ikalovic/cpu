## 六、NEXYS A7 单周期上板步骤

### 1. 新建 Vivado 工程

1. 打开 Vivado。
2. 选择 `Create Project`。
3. Project Type 选择 `RTL Project`。
4. 勾选或不勾选 `Do not specify sources at this time` 均可，建议后续手动添加文件。
5. Board 或 Part 选择 NEXYS A7 对应型号。

常见 NEXYS A7 型号：

```text
Nexys A7-100T
```

如果你的板子是 50T，应选择对应 50T 型号，并确认约束文件是否匹配。

### 2. 添加单周期源文件

添加以下 Verilog 文件：

```text
nexys-a7/sc_nexys_a7_top.v
nexys-a7/board_mem.v
nexys-a7/hex8_7seg.v

source-sc/SCCPU.v
source-sc/RF.v
source-sc/ctrl.v
source-sc/alu.v
source-sc/EXT.v
source-sc/NPC.v
source-sc/PC.v
source-sc/ctrl_encode_def.v
```

添加机器码初始化文件：

```text
source-sc/rv32_sid_sort_sim.dat
```

添加约束文件：

```text
nexys-a7/nexys_a7.xdc
```

### 3. 设置顶层模块

将顶层设置为：

```text
sc_nexys_a7_top
```

### 4. 综合、实现、生成 bitstream

依次运行：

```text
Run Synthesis
Run Implementation
Generate Bitstream
```

### 5. 下载到开发板

1. 连接 NEXYS A7。
2. 打开 `Open Hardware Manager`。
3. 点击 `Open Target`。
4. 点击 `Program Device`。
5. 选择生成的 `.bit` 文件下载。

### 6. 单周期上板观察

下载完成后：

```text
按 BTNC：复位并重新运行排序程序
SW0 = 0：数码管显示 54873530
SW0 = 1：数码管显示 03345578
LED[15:0]：显示当前数码管数值的低 16 位
```

## 八、常见问题

### 1. PowerShell 提示找不到 iverilog

说明 Icarus Verilog 没有加入 PATH。可安装 OSS CAD Suite，或修改 `build.bat` 中的路径：

```bat
set "OSS_CAD_ROOT=D:\oss-cad-suite\oss-cad-suite"
```

改为你本机 OSS CAD Suite 的实际安装目录。

### 2. 仿真输出不是 PASS

检查以下内容：

- `rv32_sid_sort_sim.dat` 是否存在。
- 是否在正确目录运行 `build.bat`。
- 是否误改了 `rv32_sid_sort_sim.asm` 或 `.dat`。
- testbench 中期望值是否仍为：

```text
0x54873530
0x03345578
```

### 3. Vivado 找不到 `rv32_sid_sort_sim.dat`

`board_mem.v` 中使用：

```verilog
$readmemh("rv32_sid_sort_sim.dat", RAM, 0, 54);
```

需要将 `.dat` 文件加入 Vivado 工程，并确保综合/仿真时它能被复制到工程运行目录。若仍无法找到，可将 `.dat` 文件复制到 Vivado 工程根目录或仿真运行目录。

### 4. 数码管不显示或显示乱码

检查：

- 顶层是否设置正确。
- 是否加入了 `nexys_a7.xdc`。
- 板子型号是否为 NEXYS A7。
- 如果是 NEXYS A7-50T，请确认所用约束和板卡版本匹配。
- 数码管段选和位选为低有效，本工程已按 NEXYS A7 连接处理。

### 5. 下载后数码管一直为 0

可尝试：

1. 按一次 `BTNC` 复位。
2. 等待一小段时间。
3. 切换 `SW0`。
4. 确认 `.dat` 文件已被 Vivado 正确加入工程。

## 九、修改为自己的学号

如果需要替换为自己的 8 位学号，需要修改汇编开头构造学号的立即数。

当前程序构造的是：

```text
54873530
```

对应汇编片段在：

```text
source-sc/rv32_sid_sort_sim.asm
source-mc/rv32_sid_sort_sim.asm
```

修改后需要：

1. 重新汇编生成新的 `rv32_sid_sort_sim.dat`。
2. 更新单周期和多周期目录中的 `.dat`。
3. 修改 `sccomp_tb.v` 和 `mccomp_tb.v` 中的原始学号、排序后学号期望值。
4. 重新运行仿真，确认 `[PASS]`。
5. 上板时更新 Vivado 工程中的 `.dat` 文件。
