## 五、代码提问准备

### 1. 单周期 CPU 可能被问

问题：单周期 CPU 的数据通路是什么？

回答：

```text
PC 取指，控制器根据 opcode/funct 生成控制信号，寄存器堆读 rs1/rs2，EXT 生成立即数，ALU 执行运算或地址计算，dm 负责 lw/sw，写回数据由 WDSel 在 ALU、MEM、PC+4 中选择，NPC 生成下一条 PC。
```

问题：为什么单周期 CPU 时钟周期长？

回答：

```text
因为每条指令必须在一个周期内完成取指、译码、执行、访存、写回，时钟周期要满足最慢指令，例如 lw 的完整路径。
```

### 2. 多周期 CPU 可能被问

问题：多周期 CPU 和单周期有什么不同？

回答：

```text
多周期把一条指令拆成 FETCH、DECODE、EXEC、MEM、WB 多个状态。每个时钟只做一部分工作，所以可以复用硬件并缩短单拍组合路径。
```

问题：多周期 CPU 为什么不需要流水线冒险处理？

回答：

```text
多周期同一时间只处理一条指令，下一条指令要等当前指令结束后才开始，因此没有多条指令重叠执行，也就没有流水线冒险。
```

### 3. 流水线 CPU 可能被问

问题：流水线 CPU 和多周期 CPU 最大区别是什么？

回答：

```text
多周期是一条指令分多拍执行，同一时间只有一条指令在 CPU 内。
流水线是多条指令同时处在不同阶段，例如一条在 EX，一条在 ID，一条在 IF。
```

问题：流水线 CPU 为什么需要冒险处理？

回答：

```text
因为多条指令重叠执行，后一条指令可能提前读取前一条尚未写回的数据，或者分支跳转导致已经取入的顺序指令无效，所以要处理数据冒险和控制冒险。
```

问题：`hazard_unit.v` 做了什么？

回答：

```text
它检测 load-use 冒险并暂停 PC 和 IF/ID，同时向 ID/EX 插入 bubble；
它根据 EX/MEM 和 MEM/WB 阶段的写回寄存器判断是否需要 forwarding；
它在 branch、jal、jalr 改变 PC 时产生 flush，清除错误取入的指令。
```

问题：`forward_a` 和 `forward_b` 是什么？

回答：

```text
它们是 ALU 两个输入端的转发选择信号。
00 表示使用 ID/EX 中寄存器堆读出的原值；
10 表示从 EX/MEM 阶段转发；
01 表示从 MEM/WB 阶段转发。
```

问题：什么是 load-use 冒险？

回答：

```text
例如 lw x1, 0(x2) 后面紧跟 add x3, x1, x4。
lw 的数据到 MEM 阶段结束才有效，而 add 在下一拍就需要进入 EX 使用 x1。
这时转发也来不及，所以需要暂停一拍。
```

问题：如何处理 `jal/jalr`？

回答：

```text
jal 和 jalr 都要把 PC+4 写回 rd 作为返回地址。
jal 的目标是 PC + J 型立即数。
jalr 的目标是 rs1 + I 型立即数，并清零最低位。
流水线中跳转在 EX 阶段确定后，要 flush 错误取入的 IF/ID 指令。
```

### 4. 学号排序程序可能被问

问题：为什么排序结果是 `03345578`？

回答：

```text
原始学号数字是 5,4,8,7,3,5,3,0。
升序排序后是 0,3,3,4,5,5,7,8。
用 BCD 十六进制显示就是 0x03345578。
```

问题：为什么检查 `dmem[96]` 和 `dmem[97]`？

回答：

```text
程序写入地址 0x180 和 0x184。
数据存储器按 32 位字寻址，用 addr[8:2] 作为下标。
0x180 / 4 = 96，0x184 / 4 = 97。
```

## 七、常见问题排查

### 1. `$readmemh` 找不到 dat 文件

现象：

```text
ERROR: $readmemh: Unable to open ...
```

处理：

```text
确认 .dat 文件在当前仿真工作目录下。
例如运行 source-pipeline 的 make run 时，dat 文件应在 source-pipeline 目录。
```

### 2. 30 条指令测试不通过

排查：

```text

```

### 3. 流水线排序不通过

优先检查：

```text
load-use 是否正确 stall。
EX/MEM 和 MEM/WB forwarding 是否正确。
branch/jal/jalr 是否 flush 错误指令。
jalr 目标地址最低位是否清零。
store 写数据是否经过 forwarding 后再进入 EX/MEM。
```

### 4. 开发板显示不对

优先检查：

```text
dat 文件是否加入 Vivado 工程。
$readmemh 文件名是否正确。
复位和开始按钮信号是否接反。
数码管段选、位选是否为低有效。
dmem[96] 和 dmem[97] 是否分别对应原始学号和排序结果。
```
