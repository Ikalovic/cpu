# 两条指令的代码级执行说明与流水线冒险处理

本文档按验收提问方式选择两条代表性指令，并逐步结合当前 Verilog 代码说明执行过程。

选择的两条指令是：

```asm
slt  x10, x1, x2     # R 型比较指令，覆盖寄存器读、ALU 比较、写回
jalr x0,  x1, 0      # I 型间接跳转，覆盖立即数、PC 修改、返回地址和冲刷
```

相关源码：

```text
source-sc/SCCPU.v
source-sc/ctrl.v
source-sc/alu.v
source-sc/EXT.v
source-sc/NPC.v
source-sc/RF.v

source-mc/MCCPU.v

source-pipeline/PipelineCPU.v
source-pipeline/pipe_decode.v
source-pipeline/pipe_alu.v
source-pipeline/pipe_branch.v
source-pipeline/hazard_unit.v
source-pipeline/RF.v
```

## 一、指令一：`slt x10, x1, x2`

`slt` 是 R 型有符号比较指令。它读取 `x1` 和 `x2`，如果 `x1 < x2`，则向 `x10` 写入 `1`，否则写入 `0`。

### 1. 单周期 CPU 中的执行过程

单周期 CPU 中，`slt` 在一个时钟周期内完成取指、译码、读寄存器、ALU 比较和写回。主要代码位于 `source-sc/SCCPU.v`、`source-sc/ctrl.v` 和 `source-sc/alu.v`。

#### 第一步：从指令中拆出字段

`SCCPU.v` 接收到 `inst_in` 后拆出指令字段：

```verilog
assign Op     = inst_in[6:0];
assign Funct7 = inst_in[31:25];
assign Funct3 = inst_in[14:12];
assign rs1    = inst_in[19:15];
assign rs2    = inst_in[24:20];
assign rd     = inst_in[11:7];
```

对 `slt x10, x1, x2` 来说，这些字段含义是：

```text
Op     = 0110011    R 型指令 opcode
Funct7 = 0000000
Funct3 = 010        表示 slt
rs1    = x1
rs2    = x2
rd     = x10
```

#### 第二步：寄存器堆读取 x1 和 x2

`SCCPU.v` 实例化寄存器堆：

```verilog
RF U_RF(
    .clk(clk), .rst(reset),
    .RFWr(RegWrite),
    .A1(rs1), .A2(rs2), .A3(rd),
    .WD(WD),
    .RD1(RD1), .RD2(RD2),
    .reg_sel(reg_sel),
    .reg_data(reg_data)
);
```

因此：

```text
A1 = x1，RD1 输出 x1 的值
A2 = x2，RD2 输出 x2 的值
A3 = x10，后续写回目标为 x10
```

#### 第三步：控制器识别 R 型 slt

`ctrl.v` 中先识别 opcode：

```verilog
7'b0110011: begin // R-type
   RegWrite = 1'b1;
   case ({Funct7, Funct3})
   {7'b0000000, 3'b010}: ALUOp = `ALUOp_slt;
   ...
   endcase
end
```

`slt` 的 `{Funct7, Funct3}` 为：

```text
{0000000, 010}
```

所以控制器输出：

```text
RegWrite = 1           允许写回 rd
ALUSrc   = 0           ALU 第二操作数来自 RD2
ALUOp    = ALUOp_slt   ALU 执行有符号小于比较
WDSel    = FromALU     写回数据来自 ALU
NPCOp    = PLUS4       下一条 PC = PC + 4
```

#### 第四步：ALU 选择两个寄存器作为输入

`SCCPU.v` 中 ALU 第二操作数选择代码为：

```verilog
assign B = (ALUSrc) ? immout : RD2;
```

因为 `slt` 是 R 型指令，`ALUSrc=0`，所以：

```text
ALU A = RD1 = x1
ALU B = RD2 = x2
```

ALU 实例化代码：

```verilog
alu U_alu(.A(RD1), .B(B), .ALUOp(ALUOp), .C(aluout), .Zero(Zero));
```

#### 第五步：ALU 执行有符号比较

`alu.v` 中对应代码：

```verilog
`ALUOp_slt: C = (A < B) ? 32'd1 : 32'd0;
```

由于 `A` 和 `B` 在模块端口中声明为 signed：

```verilog
input signed [31:0] A, B;
```

所以这里执行的是有符号比较。如果 `x1 < x2`，`aluout=1`；否则 `aluout=0`。

#### 第六步：写回 x10

`SCCPU.v` 中写回数据选择：

```verilog
always @* begin
    case(WDSel)
        `WDSel_FromALU: WD <= aluout;
        `WDSel_FromMEM: WD <= Data_in;
        `WDSel_FromPC:  WD <= PC_out+4;
        default: WD <= aluout;
    endcase
end
```

`slt` 的 `WDSel` 为 `FromALU`，所以：

```text
WD = aluout
```

寄存器堆 `RF.v` 在时钟上升沿写入：

```verilog
if (RFWr) begin
    if(A3 != 5'b0) begin
        rf[A3] <= WD;
    end
end
```

因此结果写入 `x10`。同时 `NPC.v` 默认输出 `PC+4`，CPU 顺序执行下一条指令。

### 2. 多周期 CPU 中的执行过程

多周期 CPU 的代码集中在 `source-mc/MCCPU.v`。`slt` 不在一个周期内完成，而是经过：

```text
S_FETCH -> S_DECODE -> S_EXEC -> S_WB
```

#### 第一步：S_FETCH 保存指令

`MCCPU.v` 中：

```verilog
S_FETCH: begin
    ir    <= inst_in;
    pc    <= pc + 32'd4;
    state <= S_DECODE;
end
```

这里 `ir` 保存当前 `slt` 指令。多周期 CPU 后续所有字段都从 `ir` 取：

```verilog
wire [6:0] op     = ir[6:0];
wire [6:0] funct7 = ir[31:25];
wire [2:0] funct3 = ir[14:12];
wire [4:0] rs1    = ir[19:15];
wire [4:0] rs2    = ir[24:20];
wire [4:0] rd     = ir[11:7];
```

#### 第二步：寄存器堆读 x1 和 x2

`MCCPU.v` 中寄存器堆端口：

```verilog
RF U_RF(
    .RFWr(rf_we_reg),
    .A1(rs1),
    .A2(rs2),
    .A3(rd_reg),
    .WD(wd_reg),
    .RD1(rd1),
    .RD2(rd2)
);
```

当 `ir` 是 `slt x10,x1,x2` 时：

```text
rd1 = x1
rd2 = x2
rd  = x10
```

#### 第三步：S_EXEC 调用 rtype_result

`S_EXEC` 中根据 opcode 进入 R 型分支：

```verilog
7'b0110011: begin // R-type
    aluout_reg <= rtype_result(rd1, rd2, funct7, funct3);
    state <= S_WB;
end
```

`slt` 的 `{funct7, funct3}` 为 `{0000000,010}`，因此 `rtype_result` 选择：

```verilog
{7'b0000000, 3'b010}: rtype_result = ($signed(a) < $signed(b)) ? 32'd1 : 32'd0;
```

其中：

```text
a = rd1 = x1
b = rd2 = x2
```

计算结果先保存在 `aluout_reg` 中，而不是马上写入寄存器堆。

#### 第四步：S_WB 写回 x10

写回状态中：

```verilog
S_WB: begin
    wd_reg <= (op == 7'b0000011) ? mdr : aluout_reg;
    rf_we_reg <= 1'b1;
    state <= S_FETCH;
end
```

`slt` 不是 `lw`，因此：

```text
wd_reg = aluout_reg
rf_we_reg = 1
```

目的寄存器在 `S_EXEC` 中已经保存：

```verilog
rd_reg <= rd;
```

所以 `RF.v` 在写回时把比较结果写入 `x10`。

### 3. 流水线 CPU 中的执行过程

流水线 CPU 采用多模块结构：`source-pipeline/PipelineCPU.v` 负责 PC、流水线寄存器和各模块连线；`pipe_decode.v` 负责 ID 阶段译码；`pipe_alu.v` 负责 EX 阶段运算；`pipe_branch.v` 负责分支和跳转判断；`hazard_unit.v` 负责转发、暂停和冲刷。`slt` 会依次通过 IF、ID、EX、MEM、WB 五个阶段，并通过流水线寄存器传递。

#### 第一步：IF/ID 保存取到的 slt 指令

时序逻辑中：

```verilog
if (ifid_write) begin
    ifid_valid <= 1'b1;
    ifid_pc <= pc;
    ifid_inst <= inst_in;
end
```

这表示 IF 阶段取到的 `slt` 被保存到 IF/ID 寄存器。

#### 第二步：ID 阶段由 pipe_decode.v 译码 R 型 slt

`PipelineCPU.v` 把 IF/ID 中保存的 `ifid_inst` 接到译码模块：

```verilog
pipe_decode U_DECODE(
    .inst(ifid_inst),
    .rs1(id_rs1),
    .rs2(id_rs2),
    .rd(id_rd),
    .alu_op(id_alu_op),
    .reg_write(id_reg_write),
    .uses_rs2(id_uses_rs2)
);
```

`pipe_decode.v` 内部先从 `inst` 拆字段：

```verilog
assign op     = inst[6:0];
assign funct7 = inst[31:25];
assign funct3 = inst[14:12];
assign rs1    = inst[19:15];
assign rs2    = inst[24:20];
assign rd     = inst[11:7];
```

然后在 R 型译码分支识别 `slt`：

```verilog
7'b0110011: begin
    reg_write = 1'b1;
    uses_rs2  = 1'b1;
    case ({funct7, funct3})
    {7'b0000000, 3'b010}: alu_op = `PIPE_ALU_SLT;
    {7'b0000000, 3'b011}: alu_op = `PIPE_ALU_SLTU;
    ...
    endcase
end
```

因此 `slt` 产生：

```text
id_reg_write = 1
id_uses_rs2  = 1
id_alu_op    = PIPE_ALU_SLT
```

#### 第三步：ID/EX 保存译码结果

时序逻辑把 ID 阶段结果写入 ID/EX：

```verilog
idex_valid <= ifid_valid;
idex_pc <= ifid_pc;
idex_rd1 <= rf_rd1;
idex_rd2 <= rf_rd2;
idex_rs1 <= id_rs1;
idex_rs2 <= id_rs2;
idex_rd <= id_rd;
idex_alu_op <= id_alu_op;
idex_reg_write <= id_reg_write;
```

对 `slt x10,x1,x2` 来说：

```text
idex_rd1 = x1 的值
idex_rd2 = x2 的值
idex_rd  = x10
idex_alu_op = PIPE_ALU_SLT
```

#### 第四步：EX 阶段处理可能的数据转发

流水线中，如果 `x1` 或 `x2` 刚由前面的指令产生，不能直接使用旧的 `idex_rd1/idex_rd2`。因此 `PipelineCPU.v` 先根据 `forward_a/forward_b` 选择 ALU 输入：

```verilog
case (forward_a)
2'b10: alu_in_a = exmem_alu_result;
2'b01: alu_in_a = wb_data;
default: alu_in_a = idex_rd1;
endcase

case (forward_b)
2'b10: rs2_forwarded = exmem_alu_result;
2'b01: rs2_forwarded = wb_data;
default: rs2_forwarded = idex_rd2;
endcase
```

`forward_a/forward_b` 来自 `hazard_unit.v`。这保证 `slt` 使用的是最新的 `x1` 和 `x2`。

#### 第五步：EX 阶段由 pipe_alu.v 执行有符号比较

`PipelineCPU.v` 只负责把转发后的两个操作数接入 ALU 子模块：

```verilog
pipe_alu U_ALU(
    .a(alu_in_a),
    .b(alu_in_b),
    .alu_op(idex_alu_op),
    .result(alu_result)
);
```

真正的 `slt` 运算在 `pipe_alu.v` 中：

```verilog
`PIPE_ALU_SLT: result = ($signed(a) < $signed(b)) ? 32'd1 : 32'd0;
```

其中：

```verilog
wire [31:0] alu_in_b = idex_alu_src ? idex_imm : rs2_forwarded;
```

R 型 `slt` 的 `idex_alu_src=0`，所以：

```text
alu_in_a = 最新 x1
alu_in_b = 最新 x2
```

比较结果进入 `alu_result`。

#### 第六步：EX/MEM 和 MEM/WB 传递结果

EX 阶段结果进入 EX/MEM：

```verilog
exmem_alu_result <= ex_result;
exmem_rd <= idex_rd;
exmem_reg_write <= idex_reg_write;
exmem_wb_sel <= idex_wb_sel;
```

下一拍进入 MEM/WB：

```verilog
memwb_alu_result <= exmem_alu_result;
memwb_rd <= exmem_rd;
memwb_reg_write <= exmem_reg_write;
memwb_wb_sel <= exmem_wb_sel;
```

#### 第七步：WB 阶段写回 x10

写回数据选择：

```verilog
wire [31:0] wb_data =
    (memwb_wb_sel == WB_MEM) ? memwb_mem_data : memwb_alu_result;
```

`slt` 不是 `lw`，所以写回 `memwb_alu_result`。寄存器堆写端口：

```verilog
RF U_RF(
    .RFWr(memwb_valid && memwb_reg_write),
    .A3(memwb_rd),
    .WD(wb_data)
);
```

因此最终把 `0` 或 `1` 写入 `x10`。

## 二、指令二：`jalr x0, x1, 0`

`jalr x0, x1, 0` 常用于函数返回。它的目标地址是 `x1 + 0`，并清零最低位。由于目的寄存器是 `x0`，写回返回地址会被寄存器堆忽略。

语义为：

```text
PC = (x1 + 0) & 0xfffffffe
x0 = PC + 4    # 实际写 x0 被忽略
```

### 1. 单周期 CPU 中的执行过程

#### 第一步：字段拆分

仍然由 `SCCPU.v` 拆字段：

```verilog
assign Op     = inst_in[6:0];
assign rs1    = inst_in[19:15];
assign rd     = inst_in[11:7];
assign iimm   = inst_in[31:20];
```

对 `jalr x0,x1,0`：

```text
Op  = 1100111
rs1 = x1
rd  = x0
imm = 0
```

#### 第二步：立即数扩展

`EXT.v` 中 I 型立即数扩展：

```verilog
`EXT_CTRL_ITYPE: immout <= {{20{iimm[11]}}, iimm[11:0]};
```

这里立即数为 `0`，所以 `immout=0`。

#### 第三步：控制器识别 jalr

`ctrl.v` 中：

```verilog
7'b1100111: begin // jalr
   RegWrite = 1'b1;
   ALUSrc   = 1'b1;
   EXTOp    = `EXT_CTRL_ITYPE;
   ALUOp    = `ALUOp_add;
   NPCOp    = `NPC_JALR;
   WDSel    = `WDSel_FromPC;
end
```

这些控制信号含义是：

```text
RegWrite = 1        jalr 理论上要写回 PC+4
ALUSrc   = 1        立即数参与地址计算
EXTOp    = ITYPE    生成 I 型立即数
NPCOp    = JALR     下一 PC 来自 rs1+imm
WDSel    = FromPC   写回数据选择 PC+4
```

#### 第四步：NPC 计算跳转目标

`NPC.v` 中：

```verilog
`NPC_JALR: NPC = (RS1 + IMM) & 32'hffff_fffe;
```

`SCCPU.v` 实例化时把 `RD1` 送给 `RS1`：

```verilog
NPC U_NPC(.PC(PC_out), .NPCOp(NPCOp), .IMM(immout), .RS1(RD1), .NPC(NPC));
```

因此对 `jalr x0,x1,0`：

```text
RS1 = x1
IMM = 0
NPC = x1 & 0xfffffffe
```

#### 第五步：写回 PC+4，但 x0 忽略写入

`SCCPU.v` 写回选择：

```verilog
`WDSel_FromPC: WD <= PC_out+4;
```

但是 `RF.v` 写寄存器时有保护：

```verilog
if(A3 != 5'b0) begin
    rf[A3] <= WD;
end
```

`jalr x0,x1,0` 的 `rd=x0`，即 `A3=0`，因此不会真的写入寄存器。跳转仍然发生，因为 PC 更新来自 `NPC`，不是来自寄存器写回。

### 2. 多周期 CPU 中的执行过程

`jalr` 在多周期 CPU 中的路径是：

```text
S_FETCH -> S_DECODE -> S_EXEC -> S_WB
```

#### 第一步：S_FETCH 保存 jalr 指令

`MCCPU.v`：

```verilog
S_FETCH: begin
    ir    <= inst_in;
    pc    <= pc + 32'd4;
    state <= S_DECODE;
end
```

此时 `ir` 保存 `jalr`，`pc` 已经变成原 PC+4。

#### 第二步：从 ir 得到 rs1、rd 和立即数

字段来自：

```verilog
wire [4:0] rs1 = ir[19:15];
wire [4:0] rd  = ir[11:7];
wire [31:0] imm_i = {{20{ir[31]}}, ir[31:20]};
```

对 `jalr x0,x1,0`：

```text
rs1 = x1
rd  = x0
imm_i = 0
```

#### 第三步：S_EXEC 计算返回地址和跳转目标

`MCCPU.v` 中：

```verilog
7'b1100111: begin // jalr
    aluout_reg <= pc;
    pc <= (rd1 + imm_i) & 32'hffff_fffe;
    state <= S_WB;
end
```

这里有两个关键点：

```text
aluout_reg <= pc
```

因为 `S_FETCH` 已经执行过 `pc=pc+4`，所以此时 `pc` 就是返回地址。

```text
pc <= (rd1 + imm_i) & 32'hffff_fffe
```

其中 `rd1` 是 `x1` 的值，`imm_i=0`，所以目标地址是 `x1 & 0xfffffffe`。

#### 第四步：S_WB 尝试写回，但 x0 不变

`S_WB` 中：

```verilog
wd_reg <= (op == 7'b0000011) ? mdr : aluout_reg;
rf_we_reg <= 1'b1;
```

`jalr` 不是 `lw`，所以 `wd_reg=aluout_reg=返回地址`。但是目的寄存器是 `x0`，`RF.v` 会忽略对 `x0` 的写入。因此 `jalr x0,x1,0` 最终只改变 PC，不改变寄存器。

### 3. 流水线 CPU 中的执行过程

流水线 CPU 中，`jalr` 还要处理控制冒险。因为跳转目标在 EX 阶段才确定，而 IF 阶段可能已经取入了顺序路径上的错误指令。

#### 第一步：ID 阶段由 pipe_decode.v 识别 jalr

`pipe_decode.v` 中：

```verilog
7'b1100111: begin
    imm       = imm_i;
    jalr      = 1'b1;
    alu_src   = 1'b1;
    reg_write = 1'b1;
end
```

对 `jalr x0,x1,0`：

```text
id_imm = 0
id_jalr = 1
id_alu_src = 1
id_reg_write = 1
id_rd = x0
```

#### 第二步：ID/EX 保存 jalr 控制信号

时序逻辑：

```verilog
idex_imm <= id_imm;
idex_rs1 <= id_rs1;
idex_rd <= id_rd;
idex_reg_write <= id_reg_write;
idex_alu_src <= id_alu_src;
idex_jalr <= id_jalr;
```

这使 `jalr` 到 EX 阶段时仍然带着：

```text
idex_jalr = 1
idex_rd = x0
idex_imm = 0
```

#### 第三步：EX 阶段为 rs1 做转发

`jalr` 的目标地址依赖 `rs1`。如果前一条指令刚写了 `x1`，必须使用 forwarding。`PipelineCPU.v` 中：

```verilog
case (forward_a)
2'b10: alu_in_a = exmem_alu_result;
2'b01: alu_in_a = wb_data;
default: alu_in_a = idex_rd1;
endcase
```

`alu_in_a` 就是 `jalr` 目标地址计算中的 `rs1` 值。`forward_a` 由 `hazard_unit.v` 判断：

```verilog
assign forward_a =
    (exmem_reg_write && !exmem_mem_read && (exmem_rd != 5'd0) && (exmem_rd == idex_rs1)) ? 2'b10 :
    (memwb_reg_write && (memwb_rd != 5'd0) && (memwb_rd == idex_rs1)) ? 2'b01 :
    2'b00;
```

因此 `jalr` 使用的是最新的 `x1`。

#### 第四步：EX 阶段由 pipe_branch.v 计算跳转目标

`PipelineCPU.v` 把 EX 阶段已经转发过的 `alu_in_a`、立即数和控制信号接到分支模块：

```verilog
pipe_branch U_BRANCH(
    .valid(idex_valid),
    .branch(idex_branch),
    .jal(idex_jal),
    .jalr(idex_jalr),
    .src_a(alu_in_a),
    .src_b(rs2_forwarded),
    .pc(idex_pc),
    .imm(idex_imm),
    .take_branch(ex_take_branch),
    .branch_target(ex_branch_target),
    .pc4_result(ex_pc4_result)
);
```

`pipe_branch.v` 中只要当前有效指令是 `jalr`，`take_branch=1`：

```verilog
assign take_branch = valid && ((branch && branch_ok) || jal || jalr);
```

跳转目标也在 `pipe_branch.v` 中计算：

```verilog
assign branch_target = jalr ? ((src_a + imm) & 32'hffff_fffe) :
                              (pc + imm);
```

对 `jalr x0,x1,0`：

```text
ex_branch_target = (最新 x1 + 0) & 0xfffffffe
```

#### 第五步：EX 阶段生成写回值 PC+4

`pipe_branch.v` 生成 PC+4：

```verilog
assign pc4_result = pc + 32'd4;
```

`PipelineCPU.v` 再选择跳转类指令的写回值：

```verilog
wire [31:0] ex_result = (idex_jal || idex_jalr) ? ex_pc4_result : alu_result;
```

所以 `jalr` 的写回值是 `idex_pc+4`。但本指令 `rd=x0`，最终不会改变寄存器。

#### 第六步：PC 更新并 flush 错误路径

时序逻辑中：

```verilog
if (ex_take_branch) begin
    pc <= ex_branch_target;
end
```

`hazard_unit.v` 收到 `ex_take_branch=1` 后输出：

```verilog
assign idex_flush = load_use_hazard | ex_take_branch;
assign ifid_flush = ex_take_branch;
```

`PipelineCPU.v` 清空 IF/ID：

```verilog
if (ifid_flush) begin
    ifid_valid <= 1'b0;
    ifid_pc <= 32'b0;
    ifid_inst <= 32'b0;
end
```

并通过 `idex_flush` 把 ID/EX 清成无效操作。这样 `jalr` 后面已经被错误取入的顺序指令不会继续执行。

#### 第七步：WB 阶段尝试写回 x0，但被忽略

写回端口：

```verilog
RF U_RF(
    .RFWr(memwb_valid && memwb_reg_write),
    .A3(memwb_rd),
    .WD(wb_data)
);
```

`source-pipeline/RF.v` 中同样保护 `x0`：

```verilog
if (RFWr && (A3 != 5'd0)) begin
    rf[A3] <= WD;
end
```

所以 `jalr x0,x1,0` 不会写坏 `x0`。

## 三、如何防止流水线冒险

虽然本文选择了 `slt` 和 `jalr` 两条指令，但流水线冒险处理是全局机制，所有指令都会受影响。

### 1. 普通数据冒险：forwarding

如果 `slt` 使用的 `x1` 或 `x2` 来自前一条 ALU 指令，例如：

```asm
add x1, x3, x4
slt x10, x1, x2
```

`slt` 到 EX 阶段时，`x1` 可能还没有写回寄存器。`hazard_unit.v` 用下面的代码判断是否从 EX/MEM 转发：

```verilog
(exmem_reg_write && !exmem_mem_read && (exmem_rd != 5'd0) && (exmem_rd == idex_rs1))
```

若成立：

```text
forward_a = 2'b10
```

`PipelineCPU.v` 于是选择：

```verilog
alu_in_a = exmem_alu_result;
```

这样 `slt` 使用最新结果，而不是寄存器堆旧值。

### 2. load-use 冒险：stall 和 bubble

如果出现：

```asm
lw  x1, 0(x2)
slt x10, x1, x3
```

`lw` 的数据在 MEM 阶段结束才有效，下一条 `slt` 的 EX 阶段太早，不能只靠 forwarding。`hazard_unit.v` 检测：

```verilog
assign load_use_hazard =
    idex_mem_read &&
    (idex_rd != 5'd0) &&
    ((idex_rd == id_rs1) || (id_uses_rs2 && (idex_rd == id_rs2)));
```

检测到后：

```verilog
assign pc_write   = ~load_use_hazard;
assign ifid_write = ~load_use_hazard;
assign idex_flush = load_use_hazard | ex_take_branch;
```

实际效果：

```text
PC 暂停一拍
IF/ID 保持一拍
ID/EX 插入 bubble
```

一拍之后，`lw` 的数据可以从 MEM/WB 转发给 `slt`。

### 3. 控制冒险：jalr 触发 flush

`jalr` 的目标地址在 EX 阶段才确定。此时 IF/ID 里可能已经有错误路径指令。因此 `pipe_branch.v` 中：

```verilog
assign take_branch = valid && ((branch && branch_ok) || jal || jalr);
```

该信号在 `PipelineCPU.v` 中连接为 `ex_take_branch`。对 `jalr` 来说 `ex_take_branch=1`，`hazard_unit.v` 输出：

```verilog
assign ifid_flush = ex_take_branch;
assign idex_flush = load_use_hazard | ex_take_branch;
```

因此错误路径指令被清空，不会写寄存器，也不会写内存。

## 四、验收时可直接这样讲

如果抽到 `slt`，可以按代码路径回答：

```text
SCCPU.v 先从 inst_in 拆出 Op、Funct3、Funct7、rs1、rs2、rd。
ctrl.v 看到 Op=0110011 后进入 R-type 分支，{Funct7,Funct3}=0000000_010 时设置 ALUOp_slt、RegWrite=1。
SCCPU.v 中 ALUSrc=0，所以 B=RD2，ALU 输入是 RD1 和 RD2。
alu.v 中 ALUOp_slt 执行有符号比较，成立输出 1，否则输出 0。
SCCPU.v 的 WDSel_FromALU 把 aluout 作为 WD，RF.v 在时钟上升沿写入 rd，也就是 x10。
```

如果抽到 `jalr`，可以按代码路径回答：

```text
ctrl.v 看到 Op=1100111 后设置 RegWrite=1、ALUSrc=1、EXTOp=ITYPE、NPCOp=JALR、WDSel_FromPC。
EXT.v 把 I 型立即数扩展为 immout。
NPC.v 在 NPC_JALR 分支中计算 (RS1+IMM)&0xfffffffe，RS1 来自 SCCPU.v 传入的 RD1。
SCCPU.v 的 WDSel_FromPC 把 PC_out+4 准备写回 rd。
如果 rd=x0，RF.v 因为 A3==0 会忽略写回，所以 jalr x0,x1,0 只改变 PC，不改变寄存器。
```

如果问流水线冒险：

```text
slt 的数据冒险通过 hazard_unit.v 的 forward_a/forward_b 解决，PipelineCPU.v 根据转发信号选择 EX/MEM、MEM/WB 或 ID/EX 原值，再交给 pipe_alu.v 执行比较。
load-use 情况下 hazard_unit.v 会让 pc_write=0、ifid_write=0，并用 idex_flush 插入 bubble。
jalr 的控制冒险由 pipe_branch.v 产生 ex_take_branch，再触发 ifid_flush 和 idex_flush，清掉错误路径指令。
```
